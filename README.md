# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Данные и типы данных, используемые в приложении

Интерфейс карточки товара, определяющий основные типы данных 
```
interface IProduct {
    id: string;
    category: string;
    title: string;
    image: string;
    price: number | null;
    description: string;
}
```

Интерфейс данных покупателя, определяющий основные типы данных 
```
interface IBuyer {
    payment: string;
    address: string;
    email: string;
    phone: string;
}
```

Данные карточки товара, используемые для отображения на главной странице
```
type TProductPreview = Pick<IProduct, 'category' | 'title' | 'image' | 'price'>;
```
Данные карточки товара, используемые при просмотре конкретного товара с описанием
```
type TProductDetails = Pick<IProduct, 'category' | 'title' | 'image' | 'price' | 'description'> & {buttonText:boolean};
```
Данные,  указываемые покупателем при оформлении покупки
```
type TBuyerInfo = Pick<IBuyer, 'payment' | 'address'| 'email' | 'telephone'> & { total: number; items: string[] };
```
Данные из карточки товара,  отображаемые в корзине
```
type TProductBasket = Pick<IProduct, 'title'|'price'>;
```

## Архитектура проекта

Проект построен на основе архитектуры MVP (Model-View-Presenter):

- Model (Модель) - модель для работы с  данными приложения
- View (Представление) - представление, ответственное за отображение данных пользователю.
- Presenter (Презентер) - координирует взаимодействие между моделью и представлением.

### Базовый код

#### Класс Api
Содержит в себе базовую логику отправки запросов. В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.
Методы: 
- `get` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер
- `post` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.

#### Класс EventEmitter
Брокер событий позволяет отправлять события и подписываться на события, происходящие в системе. Класс используется в презентере для обработки событий и в слоях приложения для генерации событий.  
Основные методы, реализуемые классом описаны интерфейсом `IEvents`:
- `on` - подписка на событие
- `emit` - инициализация события
- `trigger` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие   

### Слой данных

#### Класс ProductData
Класс отвечает за работу с  каталогом товаров.\
В полях класса хранятся следующие защищенные  данные:\
- `products: IProduct[]` - массив с каталогом всех имеющихся товаров
- `productItem: IProduct` - данные одной конкретной выбранной карточки товара  

Также класс содержит набор методов для работы с этими данными:
- `getProductList(): IProduct[]` - метод для получения списка всех карточек товаров для дальнейшего сохранения 
- `setProductList(products:IProduct[])` - метод для сохранения полученного списка карточек для дальшего отображения каталога на главной странице сайта.
- `getProduct(): IProduct` - метод получения  конкретного выбранного продукта из каталога для просмотра модального окна с полной информацией о товаре 
- `setProduct(productItem:IProduct)`- метод сохранения одного выбранного продукта


#### Класс BasketData
Класс для работы с корзиной выбранных товаров.\
В полях класса хранится следующие защищенные  данные:\
- `chooseProductList: IProduct[]` - массив со списком добавленных в корзину товаров

Также класс содержит набор методов для работы с этими данными:
- `addProduct(product: IProduct): void ` -  метод добавления выбранного товара в корзину
- `deleteProduct(productId:string): void` - метод удаления выбранного товара из корзины
- `showOrderList(): IProduct[]` - метод получения списка выбранных товаров в корзине
- `countProductItem(): number `- метод подсчета количества товаров, находящихся в корзине
- `countTotalAmound(): number` - метод подсчета итоговой суммы всех товаров в корзине
- `checkAviability(productId: string):boolean` - метод, проверяющий находится ли выбранный товар в корзине или нет. Если товара в корзине нет, его можно добавить в корзину; если же товар уже есть в корзине - его можно удалить из корзины. ТАкже если товар не имеет цены, он недоступен для покупки.

#### Класс BuyerData 
Класс для работы с данными покупателя на этапе оформления заказа.\
В полях класса хранится следующие данные:\
- `buyer: IBuyer = {payment, address, email, phone}` - объект данных покупателя , содержащий данные покупателя и выбранный способ оплаты
- `events: IEvents` - экземпляр класса для инициализации событий при изменении данных

Также класс содержит набор методов для работы с этими данными:
- `getBuyerInfo(): TBuyerInfo` - получение введенных данных покупателя на этапе оформления заказа
- `checkValidation(data: Record<keyof TBuyerInfo, string>): boolean `- проверка валидации введенных данных покупателя на каждом этапе ввода. Если данные валидны, переходим к следующему этапу заполнения,  если данные не валидны - выводится сообщение об ошибке.
- `setBuyerInfo(data: TBuyerInfo): void` - сохранение данных покупателя для дальнейшей отправки на сервер

### Слой представления
Все классы представления отвечают за отображение внутри контейнера (DOM-элемента) передаваемых в них данных

#### Класс Component
Класс является родительским классом  для всех компонентов представления. Класс спроектирован как универсальный шаблон, который может работать с различными типами данных. При создании экземпляра класса необходимо указать конкретный тип данных, которые будут использоваться для рендера. Через дженерик-параметр определяется тип объекта, который будет передаваться в метод `render` для отображения информации в компоненте. Конструктор класса принимает DOM-элемент (основной родительский контейнер), в котором будет размещаться компонент.
- Методы класса
    - `toggleClass` — переключает класс определенного DOM-элемента,
    - `setText` — устанавливает текст в свойство textContent определенного DOM-элемента,
    - `setDisabled` — отключает переданный DOM-элемент,
    - `setHidden` — скрывает определенный DOM-элемент,
    - `setVisible` — отображает определенный DOM-элемент,
    - `setImage` — устанавливает изображение (src) и альтернативный текст (alt) для определенного DOM-элемента,
    - `render`— возвращает контейнер, из которого элементы разметки создаются с помощью метода replaceChildren.

#### Класс Page
Наследует класс `Component`. Отвечает за отображение основных эелементов одностраничного сайта: \
- отображение иконки корзины со счетчиком количества товаров в ней. 
- отображение массива карточек товаров на главной странице сайта\
Устанавливает слушатель  на иконку корзины для открытия модального окна с корзиной товаров,  и лушатель для открытия конкретно выбранной карточки в мобальном окне для просмотра полной информации о товаре
- Конструктор класса\
`constructor(container: HTMLElement, events: IEvents)`\
принимает контейнер, в котором размещаются карточки, и брокер событий.
- Поля класса
    - `basketButton: HTMLButtonElement` - кнопка с иконкой корзины
    - `counterElement: HTMLElement` - счетчик количества выбранных товаров на кнопке корзины
    - `catalogItems: HTMLElement[]` - массив карточек товаров
    - `wrapper` - элемент для блоктровки прокрутки сайта при открытии модального окна
- Методы класса
    - `set counter(value:number)` - сеттер устанавливает количество добавленных в корзину товаров в счетчик на кнопке
    - `set catalog(items: HTMLElement[])` - сеттер устанавливает каталог товаров на главную страницу сайта
    - `set locked(value: boolean)` - сеттер проверяет наличие указанного класса в разметке 
    

#### Класс Modal
Наследует класс `Component`. Отвечает за отображение модального окна (переиспользуемый элемент), отображаемое содержимое которого будет меняться в зависимости от выбранного template. Так же предоставляет методы `open` и `close` для управления отображением модального окна. Устанавливает слушатели на клавиатуру, для закрытия модального окна по Esc, на клик в оверлей и кнопку-крестик для закрытия попапа.
- Конструктор класса\
`constructor(container: HTMLElement, events: IEvents)`\
Конструктор принимает контейнер и экземпляр класса EventEmitter для возможности инициации событий.

- Поля класса
    - `closeButton: HTMLButtonElement` - кнопка закрытия модального окна
    - `сontent: HTMLElement` - контейнер для отображения контента внутри модального окна в соответствии с выбранным шаблоном

- Методы класса
    - `set content(value: HTMLElement)` - сеттер устанавливает необходимый контент в модальном окне
    - `open()` - открытие модального окна
    - `close()` - закрытие модального окна
    - `render(data: IModal):HTMLElement` -   расширяет родительский метод - устанавливает контент внутри модального окна.

- Методы класса
    - `set title(value: string)` - сеттер устанавливает заголовок модального окна, информирующий об успешном оформлении заказа
    - `set text(value: string)`- сеттер устанавливает общую сумму оформленной покупки
    - `set button(value: string)` - сеттер устанавливает текст на кнопке возвращения к каталогу товаров

Наследует класс `Component`. Общий (родительский) класс для всех  вариантов отображения карточки товара. Отображает заголовок и цену товара, сохраняет уникальный индентификатор карточки и имеет необязательные поля: изображение товара, описание, категория.
- Конструктор класса\
`constructor(container: HTMLElement)`\
Конструктор принимает контейнер
- Поля класса
    - `title: HTMLElement` - наименование (заголовок) товара . Обязательное поле 
    - `price: HTMLElement` - стоимость товара. Обязательное поле
    - `id: HTMLElement;` - уникальный идентификатор карточки товара
    - `category?: HTMLElement` - категория товара. Не обязательное поле
    - `image?: HTMLElement` - изображение товара. Не обязательное поле 
    - `description?: HTMLElement;` - Полное описание товара. Не обязательное поле
- Методы класса
    - `get title(): string` - получает заголовок (наименование) карточки товара для установки альтернативного текста
    - `get price(): number | null` - получает стоимость товара для смены надписи на кнопках
    - `get id(): string` - получает уникальный идентификатор карточки товара
    - `set category(value:string)` - устанавливает категорию товара
    - `set image(value:string)`- устанавливает изображение товара
    - `set description(value: string)` - устанавливает описание товара

Предоставляет методы для отображения ошибок и управления активностью кнопки сабмита.
- Конструктор класса\
`constructor(container: HTMLFormElement, events: IEvents)`\
- Поля класса
    - `modalFormTitles: HTMLElement[]` - массив с заголовками для полей формы
    - `submitButton: HTMLButtonElement` - кнопка подтверждения
    - `errors: HTMLElement` - массив с ошибками валидации
- Методы класса
    - `set modalFormTitles(value:string[])` - метод устанавливает заголовки для разных полей формы
    - `set submitButton(value:string)` - метод меняет надпись на кнопке подтверждения
    - `set valid(value:boolean)` - метод проверяет валидность данных формы
    - `set errors(value:string)` - метод отображает текст ошибок в форме

Принимает в конструкторе экземпляр класса `Api` и предоставляет методы, реализующие взаимодействие с бекэндом сервиса

### Взаимодействие компонентов
Код, описывающий взаимодействие представления и данных между собой находится в файле `index.ts`, выполняющем роль презентера.\
Взаимодействие осуществляется за счет событий, генерируемых с помощью брокера событий, и обработчиков этих событий, описанных в `index.ts`\
В `index.ts` сначала создаются экземпляры всех необходимых классов, а затем настраивается обработка событий.

#### *Список всех событий, которые могут генерироваться в системе:*
##### *События изменения данных (генерируются классами моделей данных)*

- `products:update` - обновляет каталога товаров на главной странице
- `basket:update` - обновление корзины с товарами
##### *События, возникающие при взаимодействии пользователя с интерфейсом (генерируются классами, отвечающими за представление)*
- `card:select` - открытие выбранной карточка товара в модальном окне для просмотра подробной информации
- `product:buy` - добавление выбранного товара в корзину (или удаление из корзины)
- `modal:open` - открытие модального окна
- `modal:close` - закрытие модального окна
- `basket:open` - открытие модального окна с корзиной товаров
- `basket:delete` - удаление конкретного товара из корзины
- `basket:update` - обновление корзины товаров
- `formOrder:open` - событие, сообщающее о переходе к оформлению заказа
- `order:submit` - событие, создаваемое при подтверждении адреса и выборе оплаты и переход к странице с контактными данными
- `formContacts:submit` - событие, создаваемое при подтверждении имэйла и телефона
- `order.card:selected` - событие, создаваемое при выборе оплаты картой
- `order.cash:selected` - событие, создаваемое при выборе оплаты наличными
- `order.address:filled` - событие, создаваемое при вводе данных в поле адреса
- `contacts.email:filled` - событие, создаваемое при вводе данных в поле имэйла
- `contacts.phone:filled` - событие, создаваемое при вводе данных в поле телефона
- `formErrors.formOrder:filled`- событие, создаваемое для валидации формы заказа
- `formErrors.formContacts:filled`- событие, создаваемое для валидации формы с контактами
- `formOrder.success` - событие, уведомляющее об успешном оформлении заказа
